# Otus Download Videos

Инструмент для пакетного скачивания видео с платформы Otus (HLS -> MP4) по данным из `Copy as cURL`.

## Условия использования
- Используйте инструмент только при наличии действующей авторизации на Otus.
- Доступ к материалам должен быть легально получен (оплаченный/доступный курс).
- DRM-защищенный контент (Widevine/FairPlay и аналогичный) не поддерживается.

## Как работает процесс
1. Вы указываете в `config.yaml` путь к файлу с cURL-блоками (`CURL_LIST_FILE`).
2. Скрипт `add_job_from_curl.sh` пересобирает `videos.list` из этого файла.
3. Скрипт `batch_download.sh` скачивает сегменты и собирает финальные MP4.
4. При успешной сборке временные файлы задачи удаляются автоматически.
5. При ошибке логи и временные данные задачи сохраняются в `logs/` для диагностики.

## Требования
Пример для Ubuntu/Debian:

```bash
sudo apt update
sudo apt install -y bash curl ffmpeg gawk sed perl make
```

## Структура файлов
- `config.yaml` — основной конфиг проекта.
- `curl-sample.txt` — пример формата входного файла с cURL-блоками.
- `curl-list.txt` — входной файл со списком видео (можете переименовать и указать в `CURL_LIST_FILE`).
- `add_job_from_curl.sh` — генерация `videos.list` из cURL-блоков.
- `batch_download.sh` — пакетное скачивание и сборка MP4.
- `videos.list` — автогенерируемый список задач (служебный файл).
- `downloads/` — итоговые MP4.
- `logs/` — логи и временные файлы только для задач с ошибками.

## Конфигурация (`config.yaml`)
Файл уже находится в репозитории и готов к редактированию:

```yaml
CURL_LIST_FILE: curl-list.txt
BATCH_OUTPUT_DIR: downloads
BATCH_LOGS_DIR: logs
BATCH_RETRIES: 2
SKIP_EXISTING: yes
REUSE_SEGMENTS: yes
ASSEMBLE_ONLY: no
```

Описание ключей:
- `CURL_LIST_FILE` — путь к txt-файлу с блоками `имя файла + cURL`.
- `BATCH_OUTPUT_DIR` — каталог итоговых MP4.
- `BATCH_LOGS_DIR` — каталог логов и временных данных при ошибках.
- `BATCH_RETRIES` — число повторов сетевых запросов.
- `SKIP_EXISTING` — пропускать задачи, если целевой MP4 уже существует (`yes/no`).
- `REUSE_SEGMENTS` — переиспользовать уже скачанные сегменты внутри задачи (`yes/no`).
- `ASSEMBLE_ONLY` — только сборка из локально подготовленных файлов (`yes/no`).

## Формат входного файла (`CURL_LIST_FILE`)
Каждый блок:

```text
filename
curl 'https://.../media-2.ts' \
  -H 'referer: https://otus.ru/learning/...' \
  -H 'origin: https://otus.ru' \
  -H 'cookie: ...'
```

Требования к блоку:
- Первая строка — финальное имя файла (например, `filename`).
- Ниже — полный многострочный `Copy as cURL (bash)`.
- Между блоками рекомендуется пустая строка.
- `referer` должен присутствовать в заголовках cURL.

## Пошаговый запуск
1. Откройте `config.yaml` и при необходимости измените параметры.
2. Заполните файл, указанный в `CURL_LIST_FILE`, cURL-блоками.
3. Выполните основной запуск:

```bash
make run
```

## Команды Make

### `make run`
Основная рабочая команда.

Что делает:
1. Пересобирает `videos.list` из файла `CURL_LIST_FILE`.
2. Полностью перезаписывает старый `videos.list`.
3. Запускает скачивание и сборку MP4.

Когда использовать:
- Для штатного ежедневного запуска.
- После обновления `CURL_LIST_FILE`.

### `make add-list`
Только пересборка `videos.list` из `CURL_LIST_FILE` без скачивания.

Когда использовать:
- Для подготовки списка задач заранее.
- Для разделения этапов «парсинг» и «скачивание».

### `make add-list-dry`
Проверка парсинга `CURL_LIST_FILE` без записи в `videos.list`.

Когда использовать:
- Для валидации формата блоков перед запуском.
- Для быстрой проверки корректности новых cURL-вставок.

### `make batch`
Запуск скачивания по текущему `videos.list` без пересборки списка.

Когда использовать:
- Если `videos.list` уже подготовлен и его не нужно менять.

### `make batch-dry`
Проверка текущего `videos.list` без сетевых операций и без скачивания.

Когда использовать:
- Для проверки служебного списка задач перед фактическим запуском.

## Результаты и логи
- Готовые видео: `downloads/*.mp4`.
- Диагностика ошибок: `logs/<имя>__<stable_id>/job.log`.

Пример строки в консоли:

```text
[line 2][job 2/2] STEP 3/4: downloading segments...
```

Краткая расшифровка:
- `line 2` — номер строки в `videos.list`, из которой выполняется задача.
- `job 2/2` — порядковый номер задачи в текущем запуске (вторая из двух).
- `STEP 3/4` — текущий этап обработки (загрузка сегментов).

Важно:
- Для успешно завершенных задач временные файлы и логи удаляются автоматически.
- Логи остаются только для задач, завершившихся ошибкой.
- Повторный `make run` продолжает незавершенную задачу из той же папки (даже если позиция записи в `videos.list` изменилась).

## Примечания
- `videos.list` является внутренним автогенерируемым файлом. Не рекомендуется редактировать его вручную.
- Для обновления набора скачиваемых видео изменяйте только файл `CURL_LIST_FILE` и повторно запускайте `make run`.
