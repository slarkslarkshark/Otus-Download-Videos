# Otus Download Videos

Скрипт для скачивания записей лекций с платформы Otus (HLS -> MP4).

## Важные условия использования
- Инструмент предназначен только для пользователей с действующей авторизацией на Otus.
- Для доступа к материалам у Вас должен быть куплен соответствующий курс.
- DRM-контент (например, Widevine/FairPlay) данным пайплайном не поддерживается.

## Схема работы
- Автоматическая сборка `videos.list` из `curl-list.txt` (старые записи очищаются).
- Скачивание видеозаписей лекций из актуального `videos.list`.
- Поиск рабочего playlist по ссылке `media-*.ts`.
- Попытка получения ключа и скачивание сегментов.
- Сборка итогового MP4-файла.

## Предварительные требования

Пример установки для Ubuntu/Debian:

```bash
sudo apt update
sudo apt install -y bash curl ffmpeg gawk sed perl make
```

## Основные файлы проекта
- `batch_download.sh` — основной скрипт batch-скачивания и сборки.
- `add_job_from_curl.sh` — сборка `videos.list` из `curl-list.txt`.
- `curl-sample.txt` — шаблон формата для массового файла с cURL-блоками.
- `curl-list.txt` — Ваш список для массового импорта (в формате `curl-sample.txt`).
- `.env` — настройки по умолчанию.
- `videos.list` — автогенерируемый список задач.
- `downloads/` — итоговые MP4-файлы.
- `logs/` — служебные данные и логи.

## Пошаговый запуск
1. Подготовьте конфигурацию:

```bash
cp .env.example .env
```

2. Проверьте параметры в `.env`:
- `DEFAULT_REFERER` (ссылка на курс. обычно `https://otus.ru/learning/.../`)
- `DEFAULT_ORIGIN` (ссылка на платформу. обычно `https://otus.ru`)
- `DEFAULT_COOKIE` (можно оставить пустым, но при ошибках `403` или проблемах с ключом добавьте актуальный `cookie`)

3. Откройте нужный урок в браузере Otus.

4. Нажмите `F12` и перейдите на вкладку `Network`.

5. Включите видео и дождитесь появления сетевых запросов.

6. Найдите хотя бы один запрос к `*.ts` (или `media-*.ts`).

7. Нажмите правой кнопкой по запросу и выберите:
`Copy` -> `Copy as cURL (bash)`.

8. Добавьте задачу(и) в проект.

Заполните `curl-list.txt` блоками:

```text
31.mp4
curl 'https://.../media-2.ts' \
  -H 'referer: https://otus.ru/learning/...' \
  -H 'origin: https://otus.ru' \
  -H 'cookie: ...'

32.mp4
curl 'https://.../media-8.ts' \
  -H 'referer: https://otus.ru/learning/...' \
  -H 'origin: https://otus.ru'
```

Где первая строка блока — финальное имя файла, ниже — полный `Copy as cURL (bash)`.

При необходимости можно проверить парсинг без записи:

```bash
make add-list-dry
```

9. Запустите основной процесс:

```bash
make run
```

Команда `make run` делает два шага автоматически:
- полностью перезаписывает `videos.list` на основе `curl-list.txt`;
- запускает скачивание и сборку MP4.

## Полезные режимы
- Только сборка из уже скачанных локальных файлов:

```bash
ASSEMBLE_ONLY=yes make run
```

- Принудительная повторная загрузка сегментов:

```bash
REUSE_SEGMENTS=no make run
```

## Формат файла `videos.list`
Каждая строка:

```text
output_name.mp4|ts_url|referer(optional)|origin(optional)|cookie(optional)|ua=...(optional)
```

Как правило, вручную заполнять формат не требуется: `add_job_from_curl.sh` формирует строки автоматически.

## Повторный запуск без дублей
- Старые записи в `videos.list` не сохраняются при `make run`.
- Чтобы обновить набор скачивания, измените `curl-list.txt` и снова выполните `make run`.

## Где смотреть результат
- Готовые файлы: `downloads/*.mp4`
- Логи задач: `logs/<номер>_<имя>/job.log`
